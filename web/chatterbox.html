TTS · CROSS-JACOBIAN GUIDE

How fitted J-lens and cross-Jacobian diagnostics are adapted to text-to-speech

Chatterbox puts voice conditioning, input text, and earlier speech codes in one causal T3 sequence. This explorer examines one generated code at a time.

01

How it is applied

Fitted speech-code J-lensWhen a compatible artifact is loaded, it transports each intermediate speech-position residual toward the final T3 speech head and reports the fitted probability assigned to the code that was actually generated.

Per-run code→text cross-JacobianFor the selected generated code, the explorer differentiates its raw log-probability with respect to every input-text residual position at each sampled T3 layer.

Keep the methods separateThe cross-Jacobian is a local sensitivity diagnostic for the current generation—not the paper’s fitted, corpus-averaged lens.

02

What it implies

The fitted lens can show how a readout of the realized speech code changes through T3. Separately, the local trace can suggest which text-position residuals that code’s score is most sensitive to.

It does not establish

An exact word-to-waveform alignment, a phoneme label, effect direction, percent causation, or that a forced output decision is equivalent to fitted-residual steering.

03

How to use this page

  1. Choose a prepared prompt or enter a short phrase, then generate speech. The synthetic demo teaches the controls but contains fabricated values.
  2. Play the result and choose a waveform band. It selects a nominal 25 Hz speech code—not an isolated acoustic frame.
  3. Scan the fitted blue layer strips and the orange output-head strip on the same time axis as the waveform. Select a slice to compare each layer’s top acoustic code IDs and probabilities with the real output head.
  4. To test one counterfactual, select a non-realized candidate and run the separate branch action. The earlier code prefix is retained, that output decision is forced, and the later codes and waveform are regenerated.
  5. For an intermediate-state experiment, switch to Steer residual, choose one or more T3 layers, a forward speech-position span, and a norm cap. The target is not directly inserted; the result reports whether calibrated residual edits made it both raw top-1 and the processed greedy code.
  6. Then scan the blue gradient and violet attention matrices across every sampled T3 layer and tokenizer piece; these are different diagnostics.
  7. Select a text token for raw values; compare neighboring codes using Gradient L2 and the text-versus-prefix gradient-norm ratio, not only normalized colors.

Read the labels as well as the colors. Orange is the base model’s actual output-head probability. In the fitted-lens panel, blue is a separately fitted full-vocabulary readout probability. In the text-context panels, blue and violet are normalized diagnostic shares—not output probabilities.
