ASR · J-LENS GUIDE

How J-lens applies to ASR

Whisper turns audio into text through a bidirectional audio encoder and a causal text decoder. Its two lens views answer different questions.

01

How it is applied

Decoder J-lensA fitted average Jacobian carries each decoder residual toward a later decoder state, which is read through Whisper’s text head.

Encoder-to-decoder extensionA separately fitted cross-modal Jacobian maps a pooled 100 ms audio window into the final decoder’s vocabulary space.

02

What it implies

A token becoming readable in later layers can show when its lexical direction becomes accessible. An encoder peak can associate an audio location with a downstream lexical direction.

It does not establish

Emission probability, a phoneme probability, a streaming belief, conscious consideration, or causal use. Whisper’s bidirectional encoder can use surrounding and later audio.

03

How to use this page

  1. Choose a prepared sample, record, or upload audio; select the encoder overlap and analyze.
  2. Select an orange output token or a waveform position to synchronize the output head, decoder, and nearest encoder window.
  3. Compare early and late timeline rows, then hover, focus, or choose a slice to inspect token IDs, scoped ranks, and exact metric values.
  4. Use token-length filters only as vocabulary reranking aids; they do not alter generation.
  5. Check run provenance and interpretation warnings before comparing results.

Read the scales separately. Orange values are raw teacher-forced output probabilities. Blue J-lens scores are uncalibrated lexical readouts shown by display rank—not percentages or causal effect sizes.

    04 · ENCODER LENS

    Select a pooled audio window to see which downstream text-token directions become readable across encoder layers. This cross-modal view is experimental and baseline-relative.

    Blue strip: within-layer percentile of target-mean-relative logit delta · never probability

    Run an analysis with matching fitted phone prototypes to enable this view.

    What does this show?

    The nearest frozen phone prototypes for each pooled encoder J-signature, ranked by cosine similarity. Each 100 ms cell pools five native 20 ms states and shares one state with its neighbor. The label is an exploratory fitted readout—not probability, a framewise vote, a phoneme boundary, causal attribution, or a local-only receptive field.

    Why this filter?

    This phoneme-oriented view masks tokens longer than the selected limit before encoder ranking; surrounding spaces do not count. It is an Audio J-Lens exploration aid—not a phoneme classifier, model probability, or part of the original J-Lens method.

    Why only L0–L1?

    The mask is applied to the complete lexical vocabulary before ranking decoder L0 and L1. Decoder L2 and the output head stay unfiltered, so the late readout and actual model probabilities remain visible as controls. This is a display adaptation, not a change to Whisper generation or an original J-Lens feature.
